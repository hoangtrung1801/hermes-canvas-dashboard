# Docs Card Implementation Plan

> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

Goal: Add a persistent docs_card tldraw shape that stores long Markdown documents, renders them in a scrollable canvas card, provides a double-click reader side panel, and edits them through an autosaving focused modal.

Architecture: Use a dedicated typed tldraw shape with w, h, title, and content props. A Markdown-It preview module produces safe HTML, a focused modal owns local drafts and debounced commits, and the shape utility adapts both to tldraw. Creation and updates use the existing schema, CanvasBridge, live/headless executor, observations, insert menu, and auto-frame layout.

Tech stack: TypeScript 5.5, React 19, tldraw 5.2, Zod 3, Vitest 2, Testing Library, Vite, Markdown-It.

## Global Constraints

- Add a separate docs_card shape; do not change the existing native Note Card.
- Persist exactly w, h, title, and Markdown content in the shape props.
- Default size is 480 × 640; minimum size is 320 × 360; width and height resize independently.
- Preserve content source exactly, including line breaks; trim titles only.
- Support headings, paragraphs, bold/italic, links, bullet/numbered lists, blockquotes, inline code, and fenced code blocks.
- Disable raw HTML and reject unsafe links in the preview.
- Autosave modal edits with a 200ms debounce and flush before close.
- Create cards from both the insert menu and canvas.action.
- Keep live-editor and in-memory execution equivalent and include full content in observations.
- Auto-frame order is Project, Todo, Docs, Note, Link.
- Preserve all existing card, selection, movement, resizing, sync, gateway, and simulator behavior.
- Do not add a document store, upload system, attachments, search, export, history, or extended GFM features.

- [ ] Step 4: Implement:

    const markdown = new MarkdownIt({
      html: false,
      breaks: false,
      linkify: false,
      typographer: false
    })

    export function renderDocsMarkdown(content: string): DocsMarkdownResult {
      if (!content) return { html: '', error: null }
      try {
        return { html: markdown.render(content), error: null }
      } catch (error) {
        return {
          html: '',
          error: error instanceof Error ? error.message : 'Unable to render Markdown'
        }
      }
    }

Do not enable raw HTML or linkify plugins. If the installed parser does not reject unsafe schemes, provide a link validator that permits only http, https, and mailto. Keep the resulting HTML confined to the preview component’s dangerouslySetInnerHTML boundary.
